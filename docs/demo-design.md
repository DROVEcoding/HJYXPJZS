# 环境影响评价助手最小 Demo 设计

## 1. 当前目标

这个项目叫“环境影响评价助手”，目标是做一个 SaaS 网页产品的最小 demo。

本阶段不追求完整产品，而是先证明一个核心业务流程：

```text
环评老师选择项目类型 -> 系统给出环评类别辅助判断 -> 系统生成报告大纲 -> 标出缺少资料 -> 用户补充资料 -> 生成章节草稿
```

## 2. 本 Demo 的范围

本 demo 前端保留“建设项目环境影响评价分类管理名录”的整体骨架，但只实现一个真实可用分项：

```text
畜牧业 -> 生猪养殖
```

其他类目先显示“后续开放”，用于展示产品结构，不做业务生成。

## 3. 规则依据

本 demo 的规则来源以官方公开文件为主：

- 《建设项目环境影响评价分类管理名录（2021年版）》
- 《畜禽规模养殖污染防治条例》
- 《建设项目环境影响报告表》内容、格式及编制技术指南

注意：本 demo 只做“辅助判断”和“章节草稿生成”，不替代环评工程师的专业判断，也不作为法律最终结论。

## 5. 页面结构

页面采用左右结构。

左侧是环评分类骨架：

```text
建设项目环评分类管理名录（2021年版）
  ├─ 农业、林业
  ├─ 畜牧业
  │   └─ 生猪养殖（可用）
  ├─ 渔业
  ├─ 采矿业
  ├─ 制造业
  ├─ 电力、热力、燃气及水生产供应
  ├─ 交通运输、仓储、邮政
  ├─ 社会事业与服务业
  ├─ 海洋工程
  └─ 核与辐射
```

右侧是当前分项工作区。选择“生猪养殖”后显示：

1. 项目信息填写区
2. 环评类别辅助判断
3. 报告大纲
4. 缺失资料清单
5. 章节生成结果

## 6. 生猪养殖项目信息

最小 demo 先收集这些字段：

| 字段 | 中文含义 | 用途 |

| `odorControl` | 恶臭控制措施 | 用于生成大气环境影响章节 |

## 7. 环评类别辅助判断

demo 规则先做简化版：

```text
如果年出栏生猪 >= 5000，建议关注环境影响报告书；
如果存栏生猪 >= 2500 且无明确出栏量，建议关注环境影响报告书；
如果涉及环境敏感区，建议提高审慎等级；
其他规模化养殖，提示可能涉及报告表或登记表，需结合地方规定和正式名录判断。
```

页面必须显示“辅助判断”提示，避免让用户误以为系统给出了最终法律结论。

## 8. 报告大纲

最小 demo 先展示这些章节：

1. 建设项目基本情况
2. 建设内容与工程分析
3. 区域环境质量现状
4. 主要环境影响和保护措施
5. 环境保护措施监督检查清单
6. 结论

其中先支持生成 3 个章节：

- 建设项目基本情况
- 建设内容与工程分析
- 主要环境影响和保护措施

其他章节先显示“需要更多资料后续开放”。

## 9. 缺失资料逻辑

每个章节会根据用户填写情况显示缺少资料。

示例：

```text
建设项目基本情况缺少：
- 项目名称
- 建设地点
- 年出栏量或存栏量

主要环境影响和保护措施缺少：
- 粪污处理方式
- 废水处理方式
- 恶臭控制措施
```

用户补齐资料后，该章节从“资料不完整”变成“可以生成”。

## 10. 章节生成方式

最小 demo 暂时不接真实大模型。

生成方式使用：

```text
规则模板 + 用户填写资料
```

这样做的原因：

- 先验证产品流程是否顺畅。
- 降低账号、费用、密钥和后端复杂度。
- 方便后续把模板生成替换成真正智能体生成。

后续接入智能体时，前端流程可以基本保留，只替换生成章节的后端接口。

## 11. 技术架构

第一版采用静态前端 demo：

```text
index.html       页面结构
style.css        页面样式
scripts/data.js  分类、章节、字段配置
scripts/app.js   页面状态、表单、判断和生成逻辑
README.md        项目说明和文件用途表
```

暂时不做真实后端、数据库、登录和智能体服务。

原因：当前目标是最小 demo，先证明业务流程。真实 SaaS 后续再逐步加入：

1. 用户登录
2. 项目保存
3. 后端 API
4. 规则库
5. 智能体
6. 文档导出

## 12. 可能出错的地方

- 分类规则被误解为最终法律判断。
- “环境敏感区”判断过于复杂，demo 只能做简化提示。
- 生猪养殖实际还涉及地方规定、排污许可、禁养区、总量控制等内容。
- 模板生成的章节只能作为草稿，不等于合格环评报告。

## 13. 验证方式

本 demo 做完后，用这些方式验证：

1. 打开本地页面。
2. 左侧能看到完整分类骨架。
3. 点击“畜牧业 -> 生猪养殖”后，右侧出现项目信息表单。
4. 不填资料时，页面显示缺失资料。
5. 填入项目名称、地点、年出栏量等信息后，系统给出环评类别辅助判断。
6. 补齐章节所需资料后，可以生成章节草稿。
7. 其他未开放类目不会报错，会提示“后续开放”。
